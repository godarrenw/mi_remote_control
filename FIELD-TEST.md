# MiRemote 实机快速验收

目标：用 10–15 分钟确认测试包可以进入下一阶段。发现任何卡键、双触发、退出后残留映射，都先停止测试并记录当时页面与操作。

## 1. 安装与授权

- 解压 `MiRemote-Test-*.zip`，把 `MiRemote-Test.app` 拖进「应用程序」。
- 首次用右键 → 打开；按向导授予蓝牙、输入监控、辅助功能。
- 确认首页显示「已连接」和电量；向导右上角 X 可以稍后设置且下次仍会出现。
- 打开「通用 → 一键体检与修复」，除明确跳过的项目外应全部正常。

## 2. 基础 13 键

在 TextEdit 或任意安全文本框逐键测试：

- 方向键：光标正常移动，长按能连续移动，松开立即停止。
- OK：一次且仅一次 Return。
- 返回：一次且仅一次 Delete。
- 菜单：打开调度中心。
- 主页：打开 Spotlight（macOS 无启动台时的兜底）。
- TV：打开系统设置；双击进入/退出层 2。
- 音量 ±：各变化一级，无双触发。
- 语音：见下一节。
- 电源：放到最后测试，应让显示器睡眠，唤醒后 App 自动恢复。

## 3. 语音

- 豆包输入法麦克风选择 BlackHole 2ch，焦点放在文本框。
- 按住语音键说「测试一二三」，松开后应出字，语音条能正常结束。
- 连续做 5 次，目标至少成功 4 次。
- 快速短按语音键 3 次：不应卡住语音条；随后一次正常长按仍能出字。
- 语音中途关闭再打开遥控器，回连后下一次语音应恢复，无旧音频串入。

## 4. 层、手势与高级动作

- 按住 OK 超过 350ms：菜单栏显示层 1；期间上/下应调音量，松开 OK 后回基础层。
- 按住 OK 后分别按方向：上=调度中心，下=同应用窗口循环，左/右=前后标签；松开 OK 不应额外补一次 Return。
- TV 双击进入层 2：OK=Enter，返回=Esc，上下=选项，菜单=Shift+Tab，TV/音量+/音量−=数字 1/2/3；再次双击 TV 退出。
- 在 Profile 页套用一个预设，确认可预览冲突、仅填空位、撤销；导出 JSON 后再导入，配置语义保持一致。

## 5. 后台、重连与清理红线

- 关闭主窗口：App 继续运行且菜单栏图标仍在；从菜单栏重新打开设置。
- 断开遥控器再重连：状态应自动恢复，无需重启 App。
- 菜单栏选择「退出 MiRemote」：出现退出确认；确认后 App 退出。
- 退出后终端执行：

  ```bash
  hidutil property --matching '{"VendorID":0x2717,"ProductID":0x32B8}' --get 'UserKeyMapping'
  ```

  `UserKeyMapping` 必须为空，真实键盘必须完全正常。
