# UI 评审建议 Backlog

本文件保存 UI 打磨第 4 轮收口时暂不进入当前发布的建议。当前轮只处理实机阻断问题与已经落地的视觉修正；以下项目留给后续独立迭代，避免在发布前继续扩大改动面。

## P1 · 下一轮优先

- **浮层适配小屏与多显示器**：窗口选择器、系统菜单和 App 轮盘已提高字号与点击面积；后续应根据 `visibleFrame` 动态缩放或改为可滚动布局，覆盖低分辨率、副屏和显示缩放场景。验收时至少检查 1280×720 与上下左右停靠 Dock。
- **完整键盘与 VoiceOver 路径**：为遥控器示意图、录制式绑定、浮层卡片补稳定的 accessibility label/value/hint 与焦点顺序；验证仅键盘可完成新增、修改、删除和取消。
- **危险操作确认一致化**：系统功能菜单中的锁屏/睡眠等危险动作目前采用长按 OK；后续统一视觉倒计时、取消反馈和键盘等价操作，并补误触/按键松失场景测试。

## P2 · 体验增强

- **文案与术语统一**：统一“控制模式 / App 控制模式”“返回 / 取消”“授权 / 去系统设置”等用语，并为英文 README 对应界面文案预留本地化键。
- **空态与故障态插图体系**：连接中、未配对、无 Profile、无待审批会话目前以文字和系统符号为主；后续建立同一套轻量空态插图与恢复入口。
- **减少动态效果设置**：让教程脉动、轮盘/浮层转场和 toast 明确响应系统“减少动态效果”，保留状态变化但去掉位移与持续脉动。
- **截图回归自动化**：把 `docs/polish-r3` 的浅色/深色基线接入可重复的 UI preview 截图与像素差异报告，避免手工截图掩盖布局回归。

## P3 · 长期项

- **可配置外观密度**：为菜单栏面板与映射列表提供紧凑/标准密度，但不新增第三套视觉语言。
- **多语言与长文本压力测试**：覆盖英文、较长 App 名/窗口标题、超长 bundle id 和动态字体导致的截断。
- **交互遥测（本地、可关闭）**：仅在用户明确开启后记录本地匿名化操作失败计数，用于定位绑定失败、授权往返和浮层误退；不上传原始按键、App 名或文本内容。

## 已从 backlog 移出并在本轮完成

- 蓝牙授权按钮强持有 `CBCentralManager`，拒绝后改为直达系统蓝牙隐私设置。
- 每次进程启动重新检查蓝牙、输入监控、辅助功能；任一未完成时重新显示权限向导。
