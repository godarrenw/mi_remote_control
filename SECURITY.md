# 安全策略

## 支持的版本

MiRemote 目前处于早期阶段，仅对**最新发布版本**与 `main` 分支提供安全修复。

| 版本 | 是否支持 |
|---|---|
| 最新 Release | ✅ |
| 其余历史版本 | ❌ |

## 报告漏洞

本项目由个人维护，没有专职安全团队。**请不要通过公开 Issue 报告安全问题。**

请使用 GitHub 私密通报渠道：

1. 打开 [Security Advisories](https://github.com/godarrenw/mi_remote_control/security/advisories/new)
2. 提交一份私密通报，尽量包含：受影响的版本、复现步骤、影响范围、以及你认为的修复方向

我们会尽力在 **7 天内**回应，并在确认后与你协商披露时间窗口。修复发布后，如你愿意，会在
Release 说明或 CHANGELOG 中致谢。

## 值得关注的范围

MiRemote 需要几项高权限的系统能力，这些是安全反馈最有价值的地方：

- **输入监控 / 辅助功能（Accessibility）**：通过 `CGEventTap` 捕获与合成按键、读取 AX 树。
- **`hidutil` 设备级重映射**：把遥控器物理键重映射到中转键；异常退出可能残留映射（已设计自愈，
  但欢迎报告绕过场景）。
- **蓝牙 ATVV 私有 GATT 通道**：直连遥控器麦克风音频。
- **宏 / shell 动作**：允许用户配置执行外部命令，请留意任意命令执行相关的配置面风险。
- **Unix socket 批准链路**：与 Claude Code hooks 通信的本地事件通道。

## 非漏洞事项

以下为**已知且有意为之**的行为，不属于安全漏洞（详见 README FAQ）：

- App 未做 Apple 公证，首次打开需右键 → 打开；每次升级需重新授权。
- Secure Input（密码输入）期间方向键可能以中转键泄漏进前台——已在 README 标注为 v1 已知限制。
